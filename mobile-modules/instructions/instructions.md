1. Добавить в truckdoc-client-api (mobile-modules/truckdoc-common) новую схему конфигурации для инструкций, созданную из примера конфига mobile-modules/instructions/instructions_conf.json.
   За образец итогового конфига можно взять, например, имеющийся конфиг для техосмотра MaintenanceConfigInfo.
2. Сохранять новый конфиг отдельно и так же, как и образец.
3. Конфиг собой представляет дерево, где листья -файловые дескрипторы, а промежуточные и корневые узлы - фактически данные для соответствующих меню и подменю разного уровня, 
   по которым навигируешь, как по обычному файловому каталогу.
4. Узлы, находящиеся на одной высоте в дереве, должны попадать в один UI грид (табличный набор кнопок, по размеру достаточных для чтения названий из конфига и отображения иконок)
5. При нажатии на листовой узел должен открыться файл, который скачан для него и сохранен в базу, когда был получен новый конфиг. Ожидаются pdf, doc или html. 
   Открывать стандартными программами для них на телефоне.
6. Новый конфиг получается при проверке на обновление всех других конфигов, например, образца (уже есть функционал общий, нужно добавить там еще и сохранение нового конфига). 
   Новая версия всего конфига должна вызвать проверку по всем файлам в нем и уже имеющимся в базе на этот момент. 
7. Если id файла и timestamp совпадают в новом конфиге и в уже скачанном ранее и сохраненом в базу - не скачивать этот файл, 
   если нет - асинхронно скачать, с перезакачкой при разрыве соединения. 
8. Иконки для кнопок будут уже в ресурсах приложения, их скачивать не надо.
   Если имеющаяся в базе предыдущая версия файлов в новом конфиге не найдена - удалять их из базы.
